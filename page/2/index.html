Liunx 用户操作

Linux系统用户账号的管理

1、添加新的用户账号使用useradd命令，其语法如下：

# useradd 选项 用户名

选项：

  • -b，–base-dir BASE_DIR的主目录的基本目录
  • 新账户
  • -c，–comment注释新帐户的GECOS字段
  • -d，–home-dir新帐户的HOME_DIR主目录
  • -D，–defaults打印或更改默认的useradd配置
  • -e，–expiredate EXPIRE_DATE新帐户的到期日期
  • -f，–inactive新帐户的INACTIVE密码非活动期
  • -g，–gid新的主要组的组名或ID
  • 帐户
  • -G，–groups GROUPS新的补充组列表
  • 帐户
  • -h，–help显示此帮助消息并退出
  • -k，-skel SKEL_DIR使用此备用骨架目录
  • -K，–key KEY = VALUE覆盖/etc/login.defs默认值
  • -l，–no-log-init不会将用户添加到lastlog中，并且
  • 故障日志数据库
  • -m，–create-home创建用户的主目录
  • -M，–no-create-home不创建用户的主目录
  • -N，-no-user-group不会创建与以下名称相同的组
  • 用户
  • -o，–non-unique允许创建重复的用户
  • （非唯一）UID
  • -p，–password新帐户的PASSWORD密码
  • -r，–system创建系统帐户
  • -R，–root CHROOT_DIR目录以chroot进入
  • -s，–shell新帐户的SHELL登录shell
  • -u，–uid新帐户的UID用户ID
  • -U，–user-group创建一个与用户同名的组
  • -Z，–selinux-user SEUSER对SELinux用户映射使用特定的SEUSER

用户名：

# useradd –d /home/yangjrun -m yangjrun

2、修改用户密码 passwd 其语法如下

# passwd 用户名

3、修改用户的相关参数 usermod 其语法如下

# usermod [选项] 用户名

选项：

  • -c，–comment注释GECOS字段的新值
  • -d，–home HOME_DIR用户帐户的新主目录
  • -e，-expiredate EXPIRE_DATE将帐户的到期日期设置为EXPIRE_DATE
  • -f，–inactive INACTIVE设置密码过期后不活跃
  • 不活跃
  • -g，–gid GROUP强制使用GROUP作为新的主要组
  • -G，–groups GROUPS补充GROUPS的新列表
  • -a，-append将用户添加到补充GROUPS
  • -G选项提及而未删除
  • 他/她来自其他团体
  • -h，–help显示此帮助消息并退出
  • -l，–login NEW_LOGIN登录名的新值
  • -L，–lock锁定用户帐户
  • -m，–move-home将主目录的内容移动到
  • 新位置（仅与-d一起使用）
  • -o，–non-unique允许使用重复的（非唯一）UID
  • -p，–password PASSWORD使用加密的密码作为新密码
  • -R，–root CHROOT_DIR目录以chroot进入
  • -s，–shell为用户帐户新建外壳登录名
  • -u，–uid UID用户帐户的新UID
  • -U，–unlock解锁用户帐户
  • -Z，–selinux-user SEUSER用户帐户的新SELinux用户映射

4、 删除用户 userdel 其语法如下

# userdel yangjrun 删除用户

选项：

  • -f，–force强制执行某些操作，否则这些操作将失败
  • 例如删除仍登录的用户
  • 或文件，即使不是用户所有
  • -h，–help显示此帮助消息并退出
  • -r，–remove删除主目录和邮件后台处理程序
  • -R，–root CHROOT_DIR目录以chroot进入
  • -Z，–selinux-user删除该用户的任何SELinux用户映射

解决centos7使用yum安装mysql 下载速度慢的问题

1、首先备份系统自带yum源配置文件/etc/yum.repos.d/CentOS-Base.repo

# mv /etc/yum.repos.d/CentOS-Base.repo /etc/yum.repos.d/CentOS-Base.repo.backup

2、下载ailiyun的yum源配置文件到/etc/yum.repos.d/

# wget -O /etc/yum.repos.d/CentOS-Base.repo http://mirrors.aliyun.com/repo/Centos-7.repo

3、运行yum makecache生成缓存

# yum makecache

环境准备

准备vps服务器

服务器的选择需要根据个人的需求来决定：

原因：本人只需要一个梯子。

故：选择国内的服务器-阿里云-轻型应用服务器

为什么选择阿里云-轻型应用服务器

  • 价格：国外的服务器最低也是5美元一个月，这个配置是24元一个月

    国外云盘下载
  • 登录服务器(如果不懂自行百度)
  • 连接成功后，就可以复制粘贴代码部署了。
    Debian8/Debian7/Ubuntu16/Ubuntu14/CentOS7 v2ray一键部署管理脚本：

wget -N --no-check-certificate https://raw.githubusercontent.com/KiriKira/v2ray.fun/kiriMod/install.sh && bash install.sh

v2ray 配置







一键加速VPS服务器

加速教程：谷歌BBR加速教程

阿里云oss使用

默认都有阿里云账号，

创建OSS Bucket(存储空间)

登录阿里云->点击控制台->搜索 对象存储 OSS 或者 直接访问 https://oss.console.aliyun.com/overview


找到 Bucket 管理

  • 名称（随意）
  • 区域（随意）
  • 存储类型：标准存储
  • 同城冗余存储：管理
  • 版本控制：不开通
  • 读写权限：公共读（私有读取：需要对文件的链接设置有效时间，而阿里云不支持永久有效，所以设置为公共读）
  • 服务器端加密：AES256
  • 实时日志查询：不开通

测试：选中刚才创建的 Bucket -> 文件管理 -> 上传文件（选中本地的任意文件） -> 详情 -> 访问 url 地址 （如果能正常的访问 Bucket就创建好了）

开通OSS预售资源包

为什么要购买OSS预售资源包

  • 便宜 标准的本地冗余存储 40G 一年 只要 九块钱。

hexo 部署过程

发表于 2020-01-16 更新于 2020-04-28

如何使用 hexo 构建个人博客

  • 本文开发环境 windows

    环境准备

    1. git 环境
    2. nodejs环境
    3. hexo 环境

      git 环境安装

  • 前往：https://git-scm.com/download 下载 exe 文件
  • 执行 exe 文件 一直下一步就可以了。 如果需要更换安装地址,自行百度。
  • 打开 cmd 窗口 执行 git –version 测试 nodejs是否已经安装成功

    nodejs 安装

  • 前往 https://nodejs.org/zh-cn/download/ 下载 exe 文件
  • 执行 exe 文件 一直下一步就可以了。 如果需要更换安装地址,自行百度。
  • 打开 cmd 窗口 执行 npm install 测试 nodejs是否已经安装成功

    hexo 安装

  • npm install hexo –save #安装 hexo 包
  • npm install hexo-cli -g #安装 hexo 命令行模式
  • npm install hexo-deployer-git –save #hexo git 部署包

    初始化 hexo 博客并更换主题

    1. 本地初始化 hexo 模板
    2. 下载 next 主题
    3. 配置 next 主题

      本地初始化 hexo 模板

  • 创建 hexo 文件夹:为了初始化hexo
  • cd hexo 执行 hexo init

    校验:
  • hexo g/hexo generate
  • hexo s/hexo server
  • 访问 127.0.0.1:4000/localhost:4000 能成功访问hexo 初始化就没有问题了

    下载 next 主题

    • cd hexo

      git clone https://github.com/theme-next/hexo-theme-next themes/next

    • 删除 .git 文件（不删除 后面 git无法提交）
    • 修改 站点 _config.yml 文件 将 theme: -> theme: next
      校验:
    • hexo clean
    • hexo g/hexo generate
    • hexo s/hexo server
    • 访问 127.0.0.1:4000/localhost:4000 能成功访问hexo 主题就更换成功了

      配置 next 主题

      主要参考 https://github.com/iissnan/hexo-theme-next/wiki
      根据个人情况修改(应该是能看懂,文档写的很清晰)

      结合 github page 部署个人博客

    • 创建 两个仓库。

      https://github.com/yangjrun/blog (私有，用于存放 hexo 生成的项目，可以理解成源码)
      https://github.com/yangjrun/yangjrun.GitHub.io (公开，存放 hexo 编译后的静态文件，也是我的博客页面)

      TODO 这里主要是为 使用 travis-ci 自动化部署做准备的。

    • 分别执行 git clone
    • 复制 hexo 文件夹中的内容 到 blog 中
    • 修改 站点 _config.yml 文件 将 deploy 修改如下

      deploy:

      type: git

      repo: git@github.com:yangjrun/yangjrun.GitHub.io.git

      branch: master


校验:

  • hexo clean
  • hexo g
  • hexo d
  • 访问 yangjrun.GitHub.io 能成功访问 个人博客就部署完了

    结合 travis-ci 自动化部署个人博客

    • 创建 .travis.yml 文件 内容如下
      language: node_js # 指定语言
node_js:
- 12.14.1 # 指定nodejs版本
cache:
directories:
    - node_modules #执行缓存取
install: # 准备环境
- npm install hexo-deployer-git --save 
- npm install hexo --save
- npm install hexo-cli -g
- npm install
script: # 执行编译
- hexo clean
- hexo generate
after_script: # 编译之后
- git config user.name "travis ci commit"
- git config user.email "yangjrun@outlook.com"
# 替换同目录下的_config.yml文件中gh_token字符串为travis后台刚才配置的变量，注意此处sed命令用了双引号。单引号无效！
- sed -i "s/gh_token/${gh_token}/g" ./_config.yml
- hexo deploy
    • 访问 https://travis-ci.com/ 并且使用 github 账号登录。点击个人头像 进入 个人主页
      关联 blog 仓库
    • 创建 gh_token 密匙。进入 
      github->点击个人头像->setting->Developer settings->Personal access tokens
找到 generate new token 按钮 创建一个新的密匙
    • 将得到的密匙（明文只会显示一次，如果没有及时保存需要重新创建）复制。 进入 travis-ci
      找到blog项目 右上角的 more options中 setting 添加 Environment Variables
      name：gh_token value：密匙
    • 将提交 .travis.yml文件， travis-ci监测到 push。会启动构建服务。
      在job log中观察构建是否成功
